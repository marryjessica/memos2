# 待办布局优化说明

## 修改内容

优化了待办项（Memo）的显示布局，使其更加简洁和高效。

## 主要变更

### 1. **布局结构调整** (`MemoView.tsx`)
- 将布局从**垂直排列**改为**水平排列**
- 左侧：待办内容（MemoBody）- 占据主要空间
- 右侧：操作按钮区域（MemoHeader）- 固定宽度，对齐到首行
- 使用 `items-start` 确保操作按钮始终对齐到内容首行

### 2. **隐藏创建时间** (`MemoHeader.tsx`)
- 设置 `showCreator={false}` 隐藏创建者和时间信息
- 当不显示创建者时，完全隐藏左侧时间显示区域
- 容器宽度从 `w-full` 改为 `w-auto`，只占据必要空间

### 3. **操作按钮优化** (`MemoBody.tsx` 和 `MemoHeader.tsx`)
- 将反应列表（MemoReactionListView）从 MemoBody 移至 MemoHeader
- 操作按钮顺序：**表情 → 评论 → 三个点（更多操作）**
- 所有操作按钮都在右侧首行显示

## 最终效果

```
┌─────────────────────────────────────────────┬──────────────┐
│ 待办内容第一行...                              │ 😀 💬 ⋮    │
│ 待办内容第二行（如果有多行）                    │              │
│ 待办内容第三行（如果有多行）                    │              │
└─────────────────────────────────────────────┴──────────────┘
```

- **单行待办**：内容和操作按钮在同一行
- **多行待办**：操作按钮对齐到首行右侧
- **无创建时间**：更简洁的显示
